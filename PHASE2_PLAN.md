## 技术决策

### 1. 认证方案
**选择**: NextAuth.js v4 (Credentials Provider)
**理由**: 
- 已安装在 Phase 1
- 支持邮箱/密码认证
- 易于集成 Prisma Adapter

### 2. 数据库
**选择**: PostgreSQL (本地开发用 SQLite 过渡)
**理由**:
- 生产环境推荐
- 开发阶段可先用 SQLite 快速验证
- Prisma 支持无缝切换

### 3. 定价策略
**选择**: 混合制（基础订阅 + 按任务付费）
**理由**:
- 符合小老板付费习惯
- 研究团队推荐方案
- 成本结构可持续
