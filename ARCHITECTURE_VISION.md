## 🎯 产品定位

**AI Workforce Platform（AI 员工雇佣平台）**

> 类似 Upwork/Fiverr，但卖的不是人类 freelancer，而是 AI Agent

**核心体验**：
```
小老板像组建公司团队一样雇佣 AI 员工
拉进一个群，说一句话
团队自动开始工作
```

**示例场景**：
```
老板："我们要做一个手机壳品牌"
  ↓
自动组建团队：
- CEO Agent（战略规划）
- 产品经理 Agent（需求分析）
- 设计师 Agent（产品设计）
- 电商运营 Agent（店铺运营）
- 营销 Agent（推广方案）
  ↓
团队开始协作，输出完整方案
```

## 🔄 核心业务流程

### 场景 A：卖手机壳（电商运营）

```
1. 老板在 Tab1 组建"电商运营团队"
   - 雇佣：市场调研员 + 产品设计师 + 小红书运营

2. 进入 Tab2 群聊，发布任务
   "我有一批工厂资源，想做国风手机壳，
    目标是年轻女性，两周后上线预热"

3. CrewAI 引擎调度执行
   - 市场调研员：搜索爆款国风元素 → 输出报告
   - 产品设计师：基于报告 → 输出设计卖点
   - 小红书运营：基于卖点 → 输出 3 篇种草文案

4. 老板在群聊审核结果
   - 点击"同意" → 继续下一步
   - 点击"修改" → Agent 重新执行

5. 交付完整方案
   - 市场调研报告.pdf
   - 产品设计卖点.docx
   - 小红书文案 3 篇.txt
```

---

### 场景 B：开发小 App（技术开发）

```
1. 老板在 Tab1 组建"APP 开发团队"
   - 雇佣：产品经理 + 架构师 + Coder + 测试

2. 进入 Tab2 群聊，发布任务
   "我想做一个记账 App，类似鲨鱼记账"

3. CrewAI 引擎调度执行
   - 产品经理：需求分析 → PRD 文档
   - 架构师：技术选型 → 架构设计
   - Coder：编写代码 → GitHub 仓库
   - 测试：单元测试 → 测试报告

4. 关键节点人工审核 (Human-in-the-loop)
   - PRD 审核 → 老板确认
   - 架构设计 → 老板确认
   - 代码完成 → 老板确认部署

5. 交付完整 App
   - PRD 文档
   - 源代码（GitHub）
   - 部署链接（Vercel/阿里云）
```

## 💡 关键设计原则

### 1. Human-in-the-loop（人在环路）

> Agent 写完代码/文案，必须@老板审核，确认后再继续

**实现方式**：
- CrewAI 的 `human_input=True` 参数
- 群聊中的"同意"/"修改"按钮
- 关键任务节点强制审核

### 2. 工具赋能（Tooling）

> 120 个专家如果没有"工具"，就只是 120 个会聊天的闲人

**MVP 工具**：
- ✅ 网页搜索（Serper API）
- ✅ 文件读写（本地存储）
- ✅ 代码执行（沙盒环境）

**后续工具**：
- 🔜 社交媒体发布（小红书/抖音 API）
- 🔜 代码部署（Vercel/GitHub API）
- 🔜 设计生成（DALL-E/Midjourney API）

### 3. 聚焦场景（Focus）

> 不要一开始做 100 个 Agent 平台，先做一个创业团队模板

**MVP 聚焦场景**：
1. 📕 小红书营销（文案生成）
2. 📱 APP 开发（需求→代码）
3. 🛍️ 电商运营（选品→上架）

## 📊 技术架构

### 前端（OpenClaw / 小程序）

```
Next.js 16 + TypeScript + Tailwind
├── Tab1: /agents (人才大厅)
├── Tab2: /chat/[id] (群聊)
├── Tab3: /resources (资源)
└── Tab4: /settings (配置)
```

### 后端（CrewAI 引擎）

```
Python + CrewAI
├── Agent 加载器（从 OpenClaw 专家库）
├── Crew 构建器（根据团队配置）
├── Task 执行器（顺序/并行）
└── 日志输出（群聊消息）
```

### 部署

```
MVP: Vercel (前端) + Railway (Python 服务)
长期：阿里云 ECS (一体化部署)
```
